Architecture

A three-layer
covert stack

TSN routes encrypted messages through AI-generated image carriers over a peer-to-peer TCP mesh. Traffic is indistinguishable from ordinary image-sharing — no traceable metadata, no central relay.

LAYER 01 //
AES-GCM Encryption
Messages are encrypted with AES-256-GCM before any steganographic processing. A random 96-bit IV is generated per session. The GCM authentication tag guarantees integrity at the crypto layer — if decryption succeeds, the payload is authentic.
AES-256 GCM Mode 96-bit IV Auth Tag
LAYER 02 //
LSB Steganography
The ciphertext is embedded into the least-significant bits of each RGB channel in a PNG image. A 4-byte length prefix is prepended so the receiver knows exactly how many bits to extract. The visual change is perceptually invisible.
LSB Encoding RGB Channels PNG Lossless Length Header
LAYER 03 //
P2P TCP Transport
The carrier image is streamed directly between peers over a raw TCP connection. An 8-byte big-endian header encodes the exact file size. Each node runs a background listener thread, enabling bidirectional operation without a central server.
Raw TCP Size Header Multi-thread NAT Punch
LAYER 04 //
SHA-256 Integrity
A SHA-256 digest of the original plaintext is computed before encryption and returned alongside the encoded image. The receiver can recompute the hash post-decryption to verify end-to-end message integrity beyond what GCM already provides.
SHA-256 Hex Digest E2E Verify

① Encrypt — AES-256-GCM
A 96-bit random IV is generated per operation. AESGCM from cryptography.hazmat encrypts the UTF-8 encoded plaintext. The output is ciphertext + 16-byte auth tag.
# core_engine.py
aesgcm = AESGCM(key)
iv = os.urandom(12)
ciphertext = aesgcm.encrypt(iv, data, None)
↓
② Stego — LSB PNG
Payload = [4-byte length header] + [12-byte IV] + [ciphertext]. Each byte is exploded into 8 bits and written into the LSB of R, G, B channels sequentially. 1 pixel stores 3 bits.
r = (r & ~1) | int(bit_stream[bit_idx])
g = (g & ~1) | int(bit_stream[bit_idx+1])
b = (b & ~1) | int(bit_stream[bit_idx+2])
↓
③ Extract & Verify
First 32 LSBs give payload length. Then exactly that many bits are extracted, converted to bytes, split into [IV | ciphertext], decrypted, and the SHA-256 hash is verified against the original.
payload_len = struct.unpack('>I', header)[0]
iv_out = payload[:12]

Protocol Stack

How it works,
layer by layer

L1
Encryption Layer
AES-256-GCM with per-session random keys and 96-bit IVs. The GCM authentication tag makes tampering detectable before any decryption is attempted. Powered by Python's cryptography.hazmat library.
AES-256-GCM
L2
Steganography Layer
LSB (Least Significant Bit) embedding into PNG pixels via Pillow. Each pixel's RGB channels carry 3 bits. A 4-byte big-endian length prefix ensures exact extraction. Images are visually indistinguishable from unmodified carriers.
LSB PNG
L3
Transport Layer
Raw TCP socket with an 8-byte big-endian file-size header (struct format '>Q'). Receiver thread reads in 4096-byte chunks until the full payload arrives. Each node is also a server, enabling bidirectional P2P operation.
P2P TCP
L4
Integrity Layer
SHA-256 digest computed from plaintext before encryption, returned in the HTTP response header X-Plaintext-Hash. Combined with GCM's built-in auth tag, this provides dual-layer integrity verification at both cryptographic and application levels.
SHA-256
TSN

The Shadow Network is an experimental protocol combining AES-256 encryption, LSB steganography, and peer-to-peer TCP transport for metadata-resistant, censorship-proof communication.
